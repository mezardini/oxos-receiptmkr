Oxos API Documentation

Overview

The DRF API is a RESTful web service that allows clients to interact with a database of cart items. The API provides endpoints for creating, reading, updating, and deleting cart items.

Authentication

The DRF API uses token authentication to secure endpoints that modify cart items. To obtain a token, send a POST request to the /api/token/ endpoint with valid credentials. The API will respond with a token that can be used in subsequent requests by including it in the Authorization header.

Endpoints

GET /api/cartitems/

Returns a list of all cart items in the database.

POST /api/cartitems/

Creates a new cart item in the database with the provided data. The request body should be a JSON object with the following fields:

  • name: The name of the item.
  • price: The price of the item.
  • quantity: The quantity of the item.
The API will respond with a JSON object representing the created cart item.

GET /api/cartitems/{id}/

Returns the cart item with the specified ID. If the cart item does not exist, the API will respond with a 404 status code.

PUT /api/cartitems/{id}/

Updates the cart item with the specified ID with the provided data. The request body should be a JSON object with the following fields:

  • name: The name of the item.
  • price: The price of the item.
  • quantity: The quantity of the item.
The API will respond with a JSON object representing the updated cart item.

DELETE /api/cartitems/{id}/

Deletes the cart item with the specified ID. If the cart item does not exist, the API will respond with a 404 status code.
